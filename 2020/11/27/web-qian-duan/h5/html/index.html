HTML简述序言

HTML基础

1. 软件开发架构

  • C/S架构：分客户端和服务端，即可以在客户端处理请求，也可以在服务端处理。
  • B/S架构：分浏览器和服务器，只是在服务端处理请求，浏览器只是显示数据。

2. HTML基础

1>HTTP协议

Hyper Text Transfer Protocol 超文本传输协议，我们写的HTML网页运行在HTTP协议上。HTML遵守W3C规范。

2>W3C

万维网联盟，一个非盈利性组织。万维网俩能创建于1994年，是Web技术领域最具权威和影响力的国际中性技术标准机构。到目前为止，W3C已发布了200多项影响深远的Web技术标准及实施指南。

3>HTML

Hyper Text Markup Language超文本标记语言。浏览器是网页的载体，所有浏览器按照 HTML 标准进行开发，目的是解析 HTML 标准下的语义按照规则展示网页内容。HTML 标准由 W3C (World Wide Web Consortium）组织（非盈利）制定。

  • 超文本：不同于普通文本，可以描述图片，音频，视频，链接等非字符串对象。
  • 标记语言：这是一种运行在浏览器上的语言，由各种标记组成，不同于Java，没有逻辑控制。

4>HTML5

HTML5 不再代表HTML网页编程语言标准，而是围绕着HTML这个东西以及它周边的一系列网页相关技术的总称，既包含了HTML+CSS的网页制作，也包含了javascript这门编程语言的相关开发。H5 是它的简称。WEB 前端是它的别称。

5>XML和HTML的区别

个人理解：XML中的标签默认为字符串对象，只能处理字符串对象。

HMTL中的标签都是对应的一个类的对象，具有其具体的功能。我们可以认为HTML就是一份XML，这份XML的约束已经好了，例如根标签只能是<html>，<html>下面可以是<head>，之后是…然后这些标签都是对应了一个JavaBean（包括html标签），这个类具有一定的功能，例如html就是一个用来实现超文本展示的对象，head是用来展示网页的title…

HTML的格式源自XML导入了HTML的约束，HTML的约束文件编写参考JavaBean内的成员变量，这些用来实现展示数据或者布局的JavaBean响应人们的需求而诞生。版本越高的HTML自然实现了更高级的JavaBean，自然就能实现展示更高级的数据。

XML HTML
名字 eXtensible Markup Language Hyper Text Markup Language
扩展性 标签由用户自己写的<name>潘金莲</name> 不能扩展的，所有的标签都是固定，功能也是固定的。由w3c制定，如：<br/>换行
大小写敏感性 敏感<abc>与<ABC>两个不同标签 不敏感<br/>或<BR/>功能是一样，建议使用小写
功能 用于软件的配置，数据的存储 用于表示层，把数据显示给用户

3. HTML版本

  • HTML4用于电脑上比较多
  • HTML5主要用于移动端

4. HTML开发工具

  • Microsoft FrontPage 前端页面：微软的产品，目前已经淘汰。
  • Adobe Dreamweaver 织梦者：用于网页设计人员与代码编程人员。
  • HBuilder 用于网页开发工具，国产。提示功能强大。

5. HBuilder
